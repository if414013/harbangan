# Specification: Datadog APM Integration

**Track ID:** datadog-apm_20260304
**Type:** Feature
**Created:** 2026-03-04
**Status:** Draft

## Summary

Integrate Datadog Application Performance Monitoring (APM) into both the rkgw backend (Rust) and frontend (React) to provide distributed tracing, request metrics, and real-time observability via Datadog dashboards.

## Context

rkgw is a multi-user proxy gateway translating OpenAI/Anthropic API formats to Kiro. The backend already uses the `tracing` crate with `tracing_subscriber` for structured logging. This feature extends that infrastructure to ship traces and metrics to Datadog, and adds Datadog RUM to the React frontend for end-to-end visibility.

## User Story

As a platform engineer, I want distributed traces and APM metrics in Datadog so that I can monitor gateway performance, diagnose latency issues, and track error rates in production.

## Acceptance Criteria

- [ ] Backend HTTP request spans (method, path, status, duration) are visible in Datadog APM traces
- [ ] Key backend metrics (request count, latency percentiles, error rate) appear in Datadog
- [ ] Frontend page loads, route changes, errors, and user interactions are captured via Datadog RUM
- [ ] Frontend traces are connected to backend traces for end-to-end distributed tracing
- [ ] Integration is configurable via environment variables (DD_AGENT_HOST, DD_SERVICE, DD_ENV, etc.) and disabled when not set
- [ ] Datadog Agent added as an optional service in docker-compose.yml
- [ ] Log forwarding to Datadog via the DD Agent
- [ ] Works in both Full Deployment and Proxy-Only modes

## Dependencies

- Existing `tracing` + `tracing_subscriber` infrastructure in `backend/src/main.rs`
- Existing `MetricsCollector` in `backend/src/metrics/`
- Datadog account with API key (provided at deploy time)

## Out of Scope

- Custom Datadog dashboards and monitors (users create these in Datadog UI)
- Datadog Synthetic tests

## Technical Notes

- **Backend**: Use Datadog's official `datadog-opentelemetry` crate (Datadog-native, built on OTel SDK). Adds a tracing layer to the existing `tracing_subscriber::registry()` pipeline.
- **Frontend**: Use `@datadog/browser-rum-react` for React RUM integration with react-router-dom v7 route tracking.
- **DD Agent sidecar**: Added as optional Docker Compose service, receives traces via OTLP and forwards to Datadog backend.
- **Log forwarding**: Configure the DD Agent to collect container logs and forward to Datadog.
- **Graceful degradation**: When DD_AGENT_HOST is not set, no Datadog layer is added — zero overhead.
